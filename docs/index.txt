OVERVIEW
--------
HADES is a voice-first AI personal assistant for Windows. It runs as a local
Python desktop app (pywebview GUI), uses Groq LLaMA 70B for conversation, Piper
TTS for offline speech output, and Google STT for voice input. The UI is a
sci-fi dark-themed animated orb interface.

--------------------------------------------------------------------------------
WHAT HAS BEEN BUILT (complete)
--------------------------------------------------------------------------------

[CORE]
  - brain.py          Groq Llama 3.3 70B conversational AI with persistent
                      conversation history (JSON), trimmed to 20 turns, thread-safe
  - voice.py          Piper TTS (offline neural, British male voice) + fallback to
                      CLI/print; Google STT via SpeechRecognition; configurable wake
                      words (WAKE_WORDS env var) with per-word fuzzy STT-mishear
                      variant sets; 2.5s debounce (WAKE_DEBOUNCE env var);
                      MIC_ERROR sentinel distinguishes mic failure from silence;
                      listen_for_wake_word_once() helper for sleep-mode checks;
                      no per-call ambient-noise calibration in listen() (dynamic_energy
                      _threshold handles it continuously); _speak_piper_api() resamples
                      Piper's 22050 Hz output to the device's native rate (queried via
                      sd.query_devices()) and converts int16→float32 before playback to
                      eliminate Windows audio-driver stuttering
  - main.py           Full voice loop (standby → wake → listen → think → speak),
                      true sleep mode (mic on but only wake word processed; orb dims),
                      intent router (12-step priority chain), text command handler
                      with sleep-word detection (_text_state flag sets orb to SLEEPING
                      and skips standby reset; next text command wakes back to standby),
                      _pending_state multi-turn state machine for conversational flows;
                      zero-delay wake response (no artificial sleep between wake
                      detection and "Yes, Sir?" — Piper synthesis begins immediately)
  - config.py         .env loader for all API keys and settings flags, including
                      WAKE_WORDS and WAKE_DEBOUNCE
  - gui.py            pywebview window, Python↔JS bridge, status/message helpers,
                      add_help_card() for the command reference panel

[FRONTEND]
  - frontend/index.html   Single-page dark sci-fi UI — animated orb with 4 rotating
                          rings + tick marks, 5-state status system (standby /
                          sleeping / listening / thinking / speaking), scrollable
                          chat log, text input bar, help card component (.help-card)
                          with its own CSS and window.addHelpCard() JS function

[COMMANDS]
  - commands.py       Volume (set/up/down/mute via pycaw + pyautogui), time & date,
                      battery, screenshot, clipboard read, shutdown/restart/lock,
                      notes (save with optional category tag + read with category
                      filter + get_existing_categories()), timed reminders, web
                      search, app launcher (Chrome/VS Code/Notepad/Calculator/
                      Explorer/Word/Excel), website opener (YouTube/GitHub/Gmail/
                      Reddit/Twitter/Instagram/Netflix/LinkedIn), CPU/RAM/disk usage,
                      HELP_HTML constant (full command reference card)

[INTEGRATIONS]
  - weather.py        OpenWeatherMap — current weather by city
  - news.py           NewsAPI — top headlines or topic-specific news (5 results)
  - stocks.py         yfinance — live stock price; CoinGecko — crypto price
                      (BTC/ETH/SOL/DOGE/ADA/XRP)
  - spotify.py        Spotipy — play by query, pause/resume, skip, previous,
                      shuffle, current track query; HTTP-status-aware error
                      summarization (401/403/404/429/503 → clean spoken sentences)
  - vision.py         Screen capture (PIL.ImageGrab) → Groq Llama 4 Scout
                      (multimodal) — analyzes screen on voice trigger; HADES
                      persona system prompt instructs the model to describe the
                      screenshot directly as HADES (never deny screen access,
                      address user as Sir, no markdown, 2-3 sentences max)

[AUTH]
  - face_auth.py      Optional face verification gate (OpenCV + face_recognition)
                      enabled via FACE_AUTH_ENABLED=true in .env

[TESTS]

                          `pytest tests/` (no API keys, mic, or Piper model needed)

KNOWN ISSUES / BUGS
--------------------------------------------------------------------------------

  - None currently documented. File bugs here as discovered.

  FIXED THIS SESSION
  - Audio stuttering during Piper TTS playback: caused by sample-rate mismatch
    (Piper 22050 Hz vs Windows device 44100/48000 Hz) and int16 type mismatch.
    Fixed in voice.py _speak_piper_api() via device-rate query + resampling +
    int16→float32 conversion.
  - ~1 second delay between wake word detection and "Yes, Sir?" response: caused
    by an unconditional time.sleep(0.3) before Piper synthesis. Removed from
    hades_loop() in main.py; synthesis now starts immediately after wake.
